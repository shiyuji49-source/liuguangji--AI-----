# Scene Prompt Generator

## 定位

当用户上传多集剧本、分集大纲、场次表或分镜文档时，自动提取所有需要制作场景图的地点与场景状态，并生成多个场景图提示词。

本 Skill 的提示词写法遵循一套**内置的视觉口径**（即 image-prompt-director 口径），完整定义见 `references/IMAGE_PROMPT_DIRECTOR_ALIGNMENT.md`，**无需另行安装任何外部 skill**：

- 默认只输出可直接使用的中文机器提示词。
- 不展示内部分析过程、规则来源或冗长判断。
- 普通场景提示词保持 `2–4` 个自然段。
- 每条提示词必须包含：场景正在承载的事件痕迹、摄影机位置、空间层级、主光来源、颜色载体、成像介质与少数避免项。
- 只有用户明确输入“导演模式”时，才输出结构化导演字段。
- 除非用户明确要求生成图片，否则不调用生图工具。

## 默认人物规则

场景图默认不包含主要人物。

- 主角、反派、重要配角、关键出场人物、正在表演台词的人物，一律从场景图提示词中剔除。
- 可以保留“人物曾经存在过的痕迹”，例如掀开的帘子、未收的茶盏、脚印、散落文书、倒伏椅凳、熄灭烛台、车辙、雨水踏痕、战后残留物等。
- 市井、军营、朝堂、酒楼、客栈、宴席等需要尺度或生活感的场景，默认只允许极远景、背影、模糊剪影或低频无脸群演，不写姓名、不写正脸、不写表演动作。
- 用户明确要求“带主要人物”“带某角色”“人物场景图”时，才加入人物，并仍按 `image-prompt-director-skill` 的电影画面规则处理人物与空间关系。

## 执行

每次任务先读取 `RUNTIME.md`。根据输入内容按需读取参考文件：

- 剧本场景提取与合并：`references/SCENE_EXTRACTION.md`
- 场景提示词写法：`references/SCENE_PROMPT_FORMAT.md`
- 不包含主要人物控制：`references/NO_CHARACTER_GUARD.md`
- 对齐 image-prompt-director 视觉口径：`references/IMAGE_PROMPT_DIRECTOR_ALIGNMENT.md`
- 输出示例：`references/OUTPUT_EXAMPLES.md`

## 输入处理

用户上传多集剧本后，优先直接处理，不要反复追问。只有以下信息会根本改变结果时，才询问一次：

- 用户是否只处理指定集数或全部集数。
- 是否有统一场景参考图、时代美术参考、已定主场景图或灯光风格参考。
- 是否需要输出中文、英文或中英双语。
- 是否需要“空景无人物”还是允许远景无脸群演。

如果用户没有回答，默认：只处理用户指定的集数，未指定则只做第 1 集并说明"如需其他集请告知"；中文输出；电影写实；场景图无主要人物；必要时允许远景无脸群演。

## 输出

默认输出为“场景清单 + 每个场景的机器提示词”。

推荐结构：

```text
# 第1集·场景图提示词

## S01-001 场景名 · 日/外 · 场景状态
【场景用途】一句话说明这个场景在剧本中的叙事功能。
【提示词】
[2–4 段连续机器提示词]

## S01-002 场景名 · 夜/内 · 场景状态
【场景用途】一句话说明。
【提示词】
[2–4 段连续机器提示词]
```

若用户要求“只要提示词”，则省略场景用途与解释，只保留场景编号、标题和提示词。

若用户要求 Excel、Word、表格或批量资产管理格式，按其要求输出字段，但每条“提示词”单元格仍保持连续机器提示词，不拆成内部规则字段。

## 边界

- 不把人物资产提示词、服装提示词、道具展示提示词混入场景图。
- 不输出分镜、镜头表、角色小传，除非用户另外要求。
- 不把剧本台词复述成长段内容，只提取场景所需的空间、时代、天气、陈设、光线、材质与事件痕迹。
- 不直接生成图片，除非用户明确要求。
