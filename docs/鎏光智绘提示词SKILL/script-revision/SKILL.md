# 剧本医生 · 影视化剧本优化（分镜前置）

把一部多集剧本改成 **AI 能稳定生成、且下游每一条提示词都可用**的影视化版本。这是生成流水线的**最前置环节（分镜前置）**：在剧本阶段就把"会让 Seedance 提示词不可用"的隐患清掉，并建立资产档案。

## 在工作流里的位置

```
剧本医生（本 skill·分镜前置）
   ├─ 产出：影视化剧本 + 资产清单 + 环境音线索 + 诊断/改动说明
   ↓
资产提示词（人物/群演/服装/道具/场景 skill）→ image2 → 资产母版
分镜大师（静帧提示词）→ image2 + 资产 → 静帧
Seedance 视频提示词 skill → Seedance + 资产 + 静帧 → 强控视频
```

**本 skill 不拆分镜**——分镜（关键帧选取/合并）是分镜大师的活，两者并存、不冲突。本 skill 只在剧本阶段做两件事：(1) 把剧本写成"分镜大师和 Seedance 接得住"的样子；(2) 产出资产清单喂给资产 skill。

## 重要前提：这是给 AI 生成用的剧本，不是给真人剧组

判断标准从"真人好不好拍 / 成本"换成"**AI 能不能稳定生成 + 下游提示词可不可用**"。因此与"真人实拍版剧本医生"有两点根本不同：

- **不再为成本砍戏**：雨车、群演、特技、动物小孩这类真人剧组的成本约束**不适用**（`references/filmability.md` 仅当你改回真人实拍时才参考）。
- **不再为"AI 做不到"砍戏**：多人同框、复杂动作、高速运镜、画面文字——**都可以保留**，因为它们由**静帧强控**解决（image2 出好静帧 + 首尾帧锚定，Seedance 在静帧上动）。剧本阶段不要为这些发愁。

剧本阶段真正要管的，是下面准则二里那几样**静帧也救不了**的东西。

## 一句话工作流

**先通读全剧、出诊断报告 → 等用户确认修改策略 → 再分集逐集修改。** 不跳过诊断直接改，也不在确认前改完所有集。60-100 集盲改既慢又容易串戏（性格漂移、伏笔丢失、爽点错位）；先摊开问题让用户决定哪些大改/小修/别动，质量和效率都更高。

## 第零步：把剧本读进来

剧本可能是 .docx / .txt / .pdf，一个文件装全剧或一集一个文件。先统一摄入并拆集：

```bash
python scripts/extract_script.py <剧本文件或文件夹> --out /tmp/script_parsed
```

它识别格式 → 抽全文 → 按"第X集 / EP1 / 【第一集】"等切分集 → 输出 `episodes.json`（集号/标题/正文/字数/场次/日夜）并打印总览。先看 stdout 确认集数没切歪，再进入通读。无分集标记的整块文本多半是小说/大纲，先和用户确认是否"小说转剧本"。体量过大（误传长剧几十万字）改分批通读。

## 两条压倒一切的准则

### 准则一：按视听语言逻辑，不是小说逻辑（对 AI 更关键）

小说能写心理、气味、"他想起十年前"、抽象评价；**镜头只能"看见"，话筒只能"听见"。** 对 AI 尤其如此——Seedance 字面上只能渲染你描述出来的画面，"他很愤怒"必须给一个可见动作。凡观众看不见听不见的：外化成可拍的动作/表情/画面/声音，或删，或（下策）用台词/旁白点出。

常见要改的小说残留：内心独白 → 外化为动作/微表情/承载潜台词的对白；"他很善良"这类定性 → 一个能拍的具体行为；闪回回忆/嗅觉味觉/背景交代 → 画面闪回/道具暗示/冲突中带出；大段环境描写 → 提炼成 1-2 个可入镜的视觉元素。逐类改写示范见 `references/audiovisual-vs-novel.md`。

### 准则二：内容合规化 + 规避极少数生成硬伤（不是成本，也不是 AI 能力限制）

这一条管的是**静帧强控也救不了**的东西，每改一处都要过一遍：

- **内容合规（主）**——在剧本阶段就改干净，下游每条提示词才安全：
  - 血腥/尸体/伤口/酷刑 → 战损痕迹、**暗红污渍**、破损衣甲、不直视伤口的侧面构图、以光影暗示创伤。
  - 露骨/色情 → 删，或转含蓄。
  - 真实明星/公众人物 → 不拿其形象 1:1 还原，改为虚构面孔（只借气质方向）。
  - 真实品牌/商标 → 虚构品牌。
  - 危险/违法行为的直白演示 → 淡化处理。
- **生成硬伤（极少数）**：别让**关键叙事信息只靠"反射面"呈现**（镜子/水洼/刀面里的脸或字）——Seedance 一动就破坏反射几何，改成直接给到画面。
- **明确不必回避**：多人同框、复杂动作、高速运镜、画面文字——交给静帧强控，剧本阶段不要因"AI 做不到"而砍。

合规转换对照表、资产建档格式、反射规避示例见 `references/ai-adaptation.md`。

## 微短剧命门（改之前装进脑子）

竖屏微短剧靠**完播率**和**付费转化**活着：**黄金前3秒/第一集定生死**（冷开场、悬念前置）；**每集首钩尾扣**（开头钩子拉人、结尾悬念/反转钉人）；**强情绪强冲突强反转**（爽点/虐点要密、准、狠）；**付费点设计**（卡点集悬念拉满）；**信息密度**（每秒有用，废戏慢热是大忌）。更高级的处理——开放式取景做钩子、平行/交叉叙事、克制的留白与情绪烘托——用在刀刃上。分赛道套路与手法见 `references/micro-drama-craft.md`。

> **画幅按项目设定**（横屏 16:9 / 竖屏 9:16 等，与分镜大师/Seedance 保持一致），**不预设竖屏**。节奏命门与画幅无关，但具体构图（竖屏偏特写近景、横向调度受限）要看项目画幅来写。

## 分镜前置的两个交付物（喂给下游）

除了改后剧本，本 skill 还产出：

**1. 资产清单**（喂资产提示词 skill，按 Seedance 手册的"建档"格式，保证后续 @ 调用一致）：
- **固定角色档**：每个角色 = 年龄 + 身份 + 发型 + 妆容 + 服装 + 眼神气质，结尾统一加"全程固定五官、发型、穿搭不变"。
- **固定场景档**：空间 + 背景陈设 + 光线 + **环境音**（如"烛火爆裂声、远处更夫锣声"）。
- **道具 / 服装清单**：剧情关键道具与各角色服装状态。

**2. 环境音线索**：每场标 1-2 个环境音（喂 Seedance 的声场参照）。格式与建档要点见 `references/ai-adaptation.md`。

## 连贯性：制作与剪辑层面的诊断（贯穿两段）

剧本医生不只看"戏好不好"，还要替**制作**和**剪辑**提前把关连贯性——这两层问题在剧本阶段改最便宜，到了生成/剪辑阶段返工最贵：

- **制作连贯性**：AI 生成是**无状态**的——每条提示词独立出图出片，不会"记得"上一场。所以凡是跨场/跨集会**变化的状态**，剧本里必须**显式写出来**，否则下游提示词必然用错版本：
  - 角色外观状态线：服装更换点、战损/受伤/淋雨/弄脏的起止、发型妆容变化（如"自此为战损版@木兰，至第9集沐浴更衣止"）。
  - 道具状态与持有人：关键道具在谁手上、完好/破损/染血、何时交接转移。
  - 场景状态与时间线：同一场景的陈设变化（打斗后狼藉）、日夜/季节/天气连续性，跨集回到同一场景时状态是否对得上。
  - 诊断时输出**连贯性台账**（状态变化点 × 集/场），并把"一个资产的多状态版本"反馈进资产清单（如 @木兰-常服 / @木兰-战损）。
- **剪辑连贯性**：替后期预判"这两场能不能剪在一起"：
  - 场与场的衔接逻辑：动接动/静接静、出画入画方向、视线与轴线是否给够剪辑余地。
  - 转场设计：硬切是否生硬，需不需要声音先入、道具/图形匹配转场。
  - 平行/交叉叙事的交叉点是否清晰，回切时观众能否立刻定位。
  - 每集**尾钩 → 下集开头**的衔接是否顺滑，节奏曲线跨集是否断档。

## 修改的七个维度

改动归入这七类（改动说明里标注类别）。准则是"**情节大体不变，做必要微调**"，不是重写故事：

1. **情节微调**：补逻辑漏洞、钩子前置、合并拖沓、强化/补反转。伤筋动骨的大改先标出来让用户拍板。
2. **画面增删**：补能传信息/情绪的镜头（反应特写、关键道具空镜）；删冗余或与画幅不合的画面（按**剧情**和**画幅**判断，不再按"AI 做不到"删）。
3. **动作增删**：给人物加体现性格/情绪的可拍动作（外化内心）；删多余调度。
4. **台词优化**：砍过长、理顺不通、去假去尬、直白处换潜台词/一语双关；口语化、贴人物身份。要点见 `references/audiovisual-vs-novel.md`。
5. **合规与可生成优化**（原"实拍优化"）：按准则二做内容合规转换（血腥→暗红污渍等）+ 规避反射镜头。
6. **连贯性修正**：按上节台账补状态标注（角色外观/道具/场景时间线）、顺平场间衔接与转场、接好集与集的首尾。
7. **进阶手法**：冷开场/开放式取景、平行或交叉叙事、留白、情绪烘托、钩子与反转强化。用在刀刃上。

## 第一段：通读 + 诊断（出报告，然后停）

通读全剧，边读边建**剧本圣经**（分集修改的依据）：人物表（身份/动机/性格/弧光/说话风格）、主线+副线、伏笔与回收表、节奏与钩子地图（逐集标钩子/悬念/爽点/虐点/疲软段）、付费点位置、**合规红旗清单**（血腥/露骨/明星/品牌/反射镜头出现的位置）、**视听语言红旗**（小说残留位置）、**连贯性台账**（角色外观/道具/场景状态变化点 × 集/场，含剪辑衔接风险点）、以及**资产清单初稿**（角色/场景/道具/服装，含多状态版本）。

产出**诊断报告**（输出 markdown 文件）：

```
# 《剧名》剧本诊断报告（影视化 / 分镜前置）
## 一、整体判断
- 赛道与受众定位、主线清晰度、核心爽点/虐点
- 节奏总评：钩子密度、反转分布、慢热/疲软段、付费点设计、完播风险集
- 视听语言总评（小说残留严重程度）
- 合规总评（血腥/露骨/明星/品牌/反射镜头概况）
## 二、连贯性诊断（制作 + 剪辑）
- 制作连贯性台账：角色外观/道具/场景的状态变化点 × 集/场，标出剧本未显式标注的缺口
- 剪辑连贯性：场间衔接与转场风险点、平行叙事交叉点、各集尾钩→下集开头衔接评估
## 三、逐集问题清单（每集列问题 + 建议方向 + 涉及维度）
## 四、资产清单初稿（角色/场景/道具/服装，含多状态版本，供资产 skill 用）
## 五、修改策略建议（按优先级；⚠️ 任何改变情节走向的大改，逐条等用户拍板）
```

**写完报告就停，交给用户确认策略，再进入第二段。** 不自作主张开改。

## 第二段：分集修改（确认后逐集改）

带着剧本圣经和确认过的方向逐集（或按用户节奏分批）改。守住：情节大体不变只做认可的调整；时刻对照剧本圣经保证人物/伏笔/台词风格不串戏；七维度改动都服务"更可生成 + 更合规 + 更连贯 + 更抓人 + 更通畅"；输出用规范剧本格式（场景标题/场号/内外/日夜/景别、人物、对白、动作提示），格式见 `references/screenplay-format.md`。

每集（或每批）交付：**① 修改后剧本**（默认 markdown，需成片格式用 docx skill）；**② 改动说明**（按维度分组，每条"原 → 改 + 一句为什么"）；**③ 资产清单更新**（本批新增/变化的角色/场景/道具/服装档）。改完一批问用户是否继续。

```
## 第X集 改动说明
【合规与可生成优化】
- 原：刺客的脸只在刀面反光里闪过 → 改：刺客侧身入画、半张脸进光，直接给到面部
  为什么：反射镜头 Seedance 会破坏几何；改成直接呈现，静帧可锚定。
【台词优化】原："（过长台词）" → 改："（更顺/更妙）"  为什么：……
【进阶手法】开头改冷开场：先给高冲突画面再倒回起点  为什么：黄金3秒抓人，提升完播。
```
