# 鎏光机 · AI 剧制作平台 —— 完整开发规划书 v2.1（给 Claude Code）

> **用途**：把本文件放进代码仓库根目录，配合《鎏光智绘提示词SKILL》文件包，让 Claude Code 按期（P0→P1→P2）实现整个平台。
> **定位**：可对外运营的多用户 SaaS——邮箱/手机号自助注册 + **积分充值计费**，未来可给其他公司使用。
> **产品铁律（开发全程不可违背）**：
> 1. 平台 = **极薄外壳**（注册登录/角色/项目/应用注册制）+ **彼此独立的应用**；将来不断挂新应用。
> 2. **应用之间不串联**。唯一例外：剧本医生产出的"资产清单"可**一键带入**提示词生成器。其余流转由用户手动复制/下载/上传。
> 3. 各应用功能齐全（多轮对话、参数、历史、重生成），但**不做审核卡点、不做 @名强校验**。"导演已审"只是资产墙上可筛选的纯标签。
> 4. 所有 LLM 应用的"大脑" = 现成 skill 文件（§7），不重写业务逻辑。
> 5. **一切消耗都走积分**（§6）：预检余额→执行→按实际用量扣费记账。余额不足友好提示充值。

## 2. 技术选型与三方接入（已按正式文档定稿）

- **框架**：Next.js（App Router）+ TypeScript + Tailwind + shadcn/ui。
- **数据库**：PostgreSQL 16 + Drizzle ORM（连接池 max≈20）。
- **认证**：Auth.js（NextAuth v5）`credentials`。**邮箱注册**（注册→验证邮件→激活；SMTP env 配置）P0 必做；**手机号注册**可插拔模块（阿里云短信 `SMS_*`，未配置则注册页只显示邮箱）。密码 bcrypt；找回密码走邮件；用户表 `status`（运营可封号）。
- **LLM 接入（乐奇API · lqapi.top）**：用 **Anthropic 原生兼容端点 `POST https://lqapi.top/v1/messages`**（system 独立字段、max_tokens 必填、Anthropic 事件流）。实现：AI SDK `@ai-sdk/anthropic`，`baseURL=https://lqapi.top`；**鉴权 `Authorization: Bearer lq-xxx`（非默认 x-api-key，auth 头做成可切换开关）**。模型用**裸名**。**两档模型路由（env/admin 可配，按应用绑定）**：剧本医生=`claude-opus-4-8`（1M 上下文，通读长剧本刚需）；**其余一切（提示词生成器三工作区 + 会话标题等轻任务）=`claude-sonnet-4-6`**。不引入更低档模型——携带大 skill 提示词的生成对规则遵守敏感，下游视频试错成本远高于 LLM 差价。**强制：skill 系统提示词块加 `cache_control:{"type":"ephemeral"}`**（按输入/输出/缓存词元计费，命中大幅降本）；metadata 带平台用户 ID 归因；可用 `GET /v1/models` 自检密钥可见模型。**供应商切换预案**：将来可零代码切 Anthropic 官方——换 baseURL、auth 头切回 x-api-key、模型名不变；前提是服务器出口代理与海外支付（建议官方账号常备热备）。
- **图片接入（乐奇API，P1）**：模型裸名 `gpt-image-2`，同一把 `lq-` 密钥。**生成** `POST /v1/images/generations`（json：model/prompt/size/quality/n，size 支持到 4K）；**编辑/溶图** `POST /v1/images/edits`（**multipart/form-data，image 字段可重复=多参考图溶图，支持 mask 局部重绘**）。响应 `data[].url|b64_json`，部分返回 usage。计费=按张按分辨率档（§6）+ 输入词元 ¥35.35/M（编辑模式参考图计入输入）。
- **视频接入（火山方舟官方，P2）**：`ARK_API_KEY`，异步任务制（创建→轮询→取结果），实现前精读四篇官方文档：创建视频生成任务（docs/82379/1520757）、查询任务（1521309）、及 1521675、1521720。**计费已确认按 token**：`tokens = 宽×高×帧率×时长(秒) ÷ 1024`；实价 **720P=¥0.046/千 tokens（≈¥1/秒@24fps）、1080P=¥0.051/千 tokens（≈¥2.48/秒@24fps）**。**默认分辨率随项目分级：B 级=720P，S/A 级=1080P**（应用内可改）。结算按方舟任务返回的实际 usage tokens。
- **文件存储**：P1 起生成图落 `/data/assets`（Docker volume）；`STORAGE_DRIVER=local|s3` 预留。
- **长任务**：DB 任务表 + 进程内受限并发 runner + 客户端轮询（§10）。不引入 Redis。
- **部署**：国内云服务器 4C8G + Docker Compose（app/postgres/caddy）。**对外运营必须 ICP 备案**（§13）。

## 4. 平台外壳（P0）

- **注册/登录**：邮箱注册+验证激活（必做）；手机号+短信验证码（可选模块）；登录支持邮箱或手机号+密码；找回密码。
- **角色**：注册默认普通成员；平台管理员（seed 创建）在 admin 调整用户角色（导演/分镜师/美术师/后期/管理员）与状态。

- **角色→应用可见性**：导演=全部；美术=②(资产区)+③；分镜师=②(静帧/视频区)+③+④；后期=只读产物与③④成果。UI+API 双重校验。

## 6. 积分计费系统（核心，P0 落地）

**换算**：**1 元 = 100 积分**（固定）。所有价格存 `pricing_config`，admin 可改，**代码不写死单价**。

**默认定价表**（seed 值；按 1.5× 实价折算，业主上线前在 admin 核定）：
```
markup                       = 1.5      // 默认加价倍率（admin 可改）
// —— LLM（乐奇实价：opus ¥24.5/¥122.5 每百万词元；sonnet 待控制台核价，估 ¥14.7/¥73.5）
llm.sonnet.in_per_1m         = 2205     // 积分/百万输入token（估，待核价）
llm.sonnet.out_per_1m        = 11025    // （估，待核价）
llm.opus.in_per_1m           = 3675     // （成本2450）
llm.opus.out_per_1m          = 18375    // （成本12250）
llm.cached_in_ratio          = 0.1      // 缓存命中输入按 0.1× 计（以乐奇 usage 字段为准）
llm.min_per_call             = 5        // 单次最低收费
// —— 图片（乐奇实价：1K ¥0.16 / 2K ¥0.24 / 4K ¥0.32 每张 + 输入 ¥35.35/M）
image.per_1k                 = 24       // 积分/张（成本16）
image.per_2k                 = 36       // （成本24）
image.per_4k                 = 48       // （成本32）
image.input_per_1m           = 5300     // 积分/百万输入token（编辑模式参考图计入）
// —— 视频（方舟按 token：tokens=宽×高×帧率×秒÷1024；结算用任务实际 usage）
video.720p.per_1k_tokens     = 6.9      // 积分/千token（成本4.6；≈150积分/秒@24fps）
video.1080p.per_1k_tokens    = 7.65     // 积分/千token（成本5.1；≈372积分/秒@24fps）
// 内部按小数累计，最终扣费向上取整为整数积分
```

**扣费流程（`lib/billing/charge.ts` 统一入口，所有 provider 调用必经）**：
1. 预检：估算消耗（视频=请求时长×档位估价；图=张数×档位；LLM=按上限估），余额不足→拒绝并提示充值。
2. 执行 provider 调用。
3. 按**实际用量**结算：LLM 用响应 usage（含缓存命中分量）；图按实际成功张数+输入词元；视频按方舟实际 usage tokens。失败=不扣（预扣退回）。
4. 写 `credit_ledger`（带 usage 明细 ref），更新 `wallets`。**DB 事务 + 行锁保证并发正确。**

**充值**：P0=管理员手动充值（对公转账后 admin 加积分，生成 manual 订单与流水）；P1=在线支付（微信/支付宝商户号，回调入账；充值面额与赠送比存 pricing_config）。**用户钱包页**：余额/流水（每笔花在哪个应用任务）/充值入口。**admin 计费页**：定价编辑、手动充值/调整、收入与消耗报表（P2 完善）。

**参考账（写给业主，非代码）**：
- **B 级（720P）**15 秒一条：3 次提示词迭代（sonnet+缓存）≈60 积分 + 3 次抽卡 ≈6750 积分 ≈ **¥68**（成本≈¥45.6）。
- **S/A 级（1080P）**15 秒一条：3 次抽卡 ≈16740 积分 ≈ **¥167**（成本≈¥112）——1080P 是 720P 的 2.5 倍，**静帧强控的价值随之翻倍**：一张 2K 静帧售 36 积分（成本¥0.24），若让 1080P 少抽一次卡即省 5580 积分（成本省¥37），百倍杠杆。S/A 多出静帧在账面上稳赚。
- 视频永远是成本与定价的绝对大头；定价杠杆主要调 video.* 两行。

## 7. skill → 系统提示词

启动时把 `docs/鎏光智绘提示词SKILL/<文件夹>/` 的 SKILL.md（+RUNTIME.md）+ references 全量拼接缓存。映射：`script-doctor`→剧本医生；`prompt-studio:人物|服装|道具|场景|群演`→对应资产 skill；`prompt-studio:静帧`→分镜大师；`prompt-studio:视频`→Seedance视频提示词。运行时附注注入【项目分级】【画幅】【当前集】。换 skill=换文件重启。
剧本分集：上传 .docx/.txt/.pdf → 服务端抽文本（mammoth / pdf-parse）→ 交给 LLM 按 skill 规则切，不跑 Python 脚本。

## 8. 应用①剧本医生 & ②提示词生成器（P0，对话式）

共用会话框架：流式 useChat、会话列表、快捷参数栏、消息可复制/存产物/重生成/追问、参考图多模态输入。①参数=集数；模型=opus-4-8。②三工作区（资产 5 模式/静帧/视频）+ 参数（模式/集数/分级/画幅/参考图）+ **「带入资产清单」为全站唯一跨应用带入**；模型=sonnet-4-6。每次发送前走 §6 预检；每条 assistant 消息底部显示本次消耗积分（hover 看 token 与缓存明细）。

## 9. 应用③ image2 图像生成器（P1）

参考 alasano/gpt-image-playground 形态：生成/编辑（多图溶图+蒙版）双模式、画幅预设（3:4 / 4:5 / 16:9 / 9:16 / 自定义至 4K）、质量档、批量 1–4 张、资产墙（kind/@名/集数/已审筛选、详情含提示词参数、同参数重生成）。接入按 §2 图片定稿（generations=json，edits=multipart 多图+mask）；按档位×张数+输入词元扣费，失败退回；卡片显示消耗。

## 10. 任务机制 & 应用④ Seedance 视频生成器（P2）

任务机制：gen_tasks + 进程内受限并发 runner（`OPENAI_CONCURRENCY`/`ARK_CONCURRENCY` env）+ 客户端轮询；失败可一键重试。
应用④：多图参考选择器（从资产墙勾选+拖拽排序=image1/2/…handle 顺序；可临时上传）；提示词手输/粘贴；参数=画幅（默认随项目分级：B=720P、S/A=1080P，可改）/时长 4–15s/模型档；接方舟官方异步 API（创建→轮询→成片落盘入库→站内播放/下载）；详情留"提示词+参考图组合+参数"可复现重跑；任务中心一览（状态/耗时/成本/重试）。扣费=方舟实际 usage tokens × 档位单价，失败全额退回；预检按请求时长预估预扣、结算多退少补扣。

## 11. 多公司使用

P0–P2 用"项目"即可服务外部团队（建项目拉人）。不做组织/租户层（避免过度设计）；将来有企业批量开户需求再加 org 层（projects 预留 org_id 列）。公司客户充值用 admin 手动充值到其指定账号。

## 12. 设计系统

暗色影视级：背景 #0E0F12 / 面板 #16181D / 描边 #262A33 / 正文 #E8E6E1；唯一点缀色鎏光金 #C9A86A（hover #E0C589）；MiSans/思源黑体细字重；等宽数字；圆角 10px；资产墙瀑布卡片；动效 150–200ms 微过渡。气质=剪辑软件+片场资料墙，不像 OA。钱包/流水/充值/admin 计费页沿用同一语言。

## 13. 部署与合规

1. 国内云服务器 4C8G + 200GB 数据盘，Ubuntu 22.04 + Docker Compose（app/postgres/caddy 自动 HTTPS）。
2. `.env`：`DATABASE_URL`、`AUTH_SECRET`、`LLM_BASE_URL=https://lqapi.top`、`LLM_API_KEY`、`LLM_MODEL_HEAVY=claude-opus-4-8`、`LLM_MODEL_MAIN=claude-sonnet-4-6`、`OPENAI_BASE_URL=https://lqapi.top/v1`、`OPENAI_API_KEY`(P1)、`ARK_API_KEY`(P2)、`SMTP_*`、可选 `SMS_*`、并发数。密钥只在服务器。
3. **合规（对外运营必做）**：域名 **ICP 备案**；微信/支付宝官方商户需企业资质；用户协议与隐私政策页（注册勾选）；内容投诉入口。
4. 备份：pg_dump 每日 + /data/assets 快照。容量不够先扩盘或切 s3。

5. 每次 LLM 调用按实际 token（含缓存分量）扣费且消息底部可见；**抽查二次调用确有缓存命中、费用显著低于首调**；余额不足被预检拦截。

**P1**
