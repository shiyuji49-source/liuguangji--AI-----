# 凭证清单（只记"是什么、在哪用"，不存值！）

> **规则：值只存两个地方**——服务器/本机的 `.env`（已 gitignore）+ 密码管理器（推荐 Bitwarden 免费版或 1Password，每个服务一条记录，备注里写对应的 env 变量名）。
> 任何 `.md`/聊天记录/截图里都不要出现真实密钥。忘了就去密码管理器查，泄露了按下表的"轮换方式"换新。

## 一、env 变量总表（.env，模板见 .env.example）

| 变量 | 是什么 | 在哪申请/查看 | 泄露后轮换方式 |
|---|---|---|---|
| `DATABASE_URL` | PostgreSQL 连接串 | 本地=embedded(:5433)；线上=自建/云数据库控制台 | 改库密码后更新连接串 |
| `AUTH_SECRET` | NextAuth 会话签名密钥 | `openssl rand -base64 32` 生成 | 重新生成（全员需重新登录） |
| `LLM_BASE_URL` | LLM API 地址 | 当前=乐奇 lqapi.top；可换 Kimi/官方 | 改地址即可 |
| `LLM_API_KEY` | LLM API 密钥 | 乐奇控制台 / moonshot 控制台 | 控制台重置 key |
| `LLM_AUTH_STYLE` | 鉴权方式（bearer/x-api-key） | 跟随服务商文档 | — |
| `LLM_MODEL_HEAVY` / `LLM_MODEL_MAIN` | 模型名 | 服务商模型列表 | — |
| `ARK_API_KEY` | 火山方舟（Seedance/即梦，P2 用） | 火山引擎控制台 | 控制台重置 |
| `KIMI_BASE_URL/API_KEY/MODEL` | Kimi 官方 API（Anthropic 兼容端点 api.moonshot.cn/anthropic，候选主模型） | platform.kimi.com 控制台 | 控制台重置 key |
| `SMTP_HOST/PORT/USER/PASS` | 注册验证邮件 | 企业邮箱/阿里云邮件推送 | 邮箱后台重置授权码 |
| `SEED_ADMIN_EMAIL/PASSWORD` | 种子管理员 | 自定（首次 db:seed 用） | 登录后改密码 |

## 三、习惯建议（治"经常混乱忘记"）

1. **单一真相源**：值只在 `.env` + 密码管理器；本文件只当"地图"。
2. **新增任何凭证时**：先在本表加一行 → 值进密码管理器 → 写进 `.env` → `.env.example` 加占位行。三步做完才算接入。
3. **命名规律**：env 变量名 = 服务名_用途（`LLM_API_KEY`、`ARK_API_KEY`），密码管理器条目同名，搜索即达。
4. **每季度轮换**一次外部 API key（控制台重置 → 更新 .env → 重启服务）。
5. ⚠️ 项目里如有散落的明文密钥文件（如桌面上的 `API key.md`），把值迁入密码管理器后**删除文件**。
