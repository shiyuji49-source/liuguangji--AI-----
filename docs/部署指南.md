# 鎏光机内测部署指南（小白版，照着敲就能上线）

> 目标：花 1-2 小时，把鎏光机跑在一台云服务器上，公司同事用浏览器输入 `http://服务器IP:3000` 就能用。
> 全程只需要会复制粘贴命令。`$` 开头的是要在终端里敲的命令（不要敲 $ 本身）。

---

## 第 0 步：买一台云服务器（10 分钟）

推荐**腾讯云·轻量应用服务器**（或阿里云·轻量应用服务器，流程几乎一样）：

1. 打开 [腾讯云轻量服务器](https://cloud.tencent.com/product/lighthouse)，登录/注册，实名认证。
2. 选购配置：
   - **地域**：离公司近的（如上海/广州）
   - **镜像**：选「**Ubuntu 22.04**」（系统镜像，不要选应用镜像）
   - **套餐**：**2 核 4G、60GB SSD** 起步（约 ¥70-110/月，内测够用；人多再升）
3. 下单后在控制台「重置密码」给 root 用户设一个强密码 → **存进密码管理器**。
4. 控制台找到服务器的**公网 IP**（形如 119.x.x.x），记下来。
5. **防火墙放行端口**：控制台 →「防火墙」→ 添加规则：放行 **TCP 3000**（应用）；22（SSH）默认已放行。

> 备案说明：用 `IP:端口` 访问**不需要备案**。以后想绑域名+国内服务器才需要 ICP 备案（或用香港节点免备案）。

## 第 1 步：连上服务器（5 分钟）

在你的 Mac 上打开「终端」：

```
$ ssh root@你的公网IP
```

第一次连接输 `yes`，然后输你设的 root 密码（输入时屏幕不显示，正常）。看到 `root@xxx:~#` 就是连上了。**之后所有命令都在这个 SSH 窗口里敲。**

## 第 2 步：装 Docker（5 分钟）

```
$ curl -fsSL https://get.docker.com | sh
$ systemctl enable --now docker
$ docker --version
```

最后一条能打印版本号（如 `Docker version 27.x`）就成功了。

> 国内服务器拉镜像慢的话，配置镜像加速（腾讯云内网加速器）：
> ```
> $ mkdir -p /etc/docker && cat > /etc/docker/daemon.json <<'EOF'
> { "registry-mirrors": ["https://mirror.ccs.tencentyun.com"] }
> EOF
> $ systemctl restart docker
> ```

## 第 3 步：把代码传上服务器（10 分钟）

**方式 A（推荐）— Git 私有仓库**：先在本地把代码推到一个**私有** GitHub/Gitee 仓库，然后服务器上：

```
$ apt install -y git
$ git clone https://你的私有仓库地址.git /opt/liuguangji
$ cd /opt/liuguangji
```

**方式 B — 直接打包上传**（不想弄 Git 时）：在你 Mac 的另一个终端窗口（不是 SSH 窗口）：

```
$ cd "/Users/wujiaodaloushiwozhade/Desktop/鎏光机/鎏光机 · AI 剧制作平台"
$ tar czf /tmp/liuguangji.tar.gz --exclude node_modules --exclude .next --exclude .env liuguangji
$ scp /tmp/liuguangji.tar.gz root@你的公网IP:/opt/
```

然后回到 SSH 窗口：

```
$ cd /opt && tar xzf liuguangji.tar.gz && cd liuguangji
```

## 第 4 步：写生产环境 .env（10 分钟，最重要的一步）

```
$ cp .env.example .env
$ nano .env
```

（nano 是个编辑器：方向键移动，改完按 `Ctrl+O` 回车保存，`Ctrl+X` 退出）

把下面这些**逐项填好**（参考 docs/凭证清单.md，值从你的密码管理器拿）：

```ini
# 数据库（compose 自动用，只需设一个强密码）
DB_PASSWORD=一个强密码

# 会话密钥（在服务器上跑 openssl rand -base64 32 生成一个新的，别用本地那个）
AUTH_SECRET=新生成的随机串

# LLM（当前主模型 Kimi）
LLM_BASE_URL=https://api.moonshot.cn/anthropic
LLM_API_KEY=你的Kimi key
LLM_AUTH_STYLE=bearer
LLM_MODEL_MAIN=kimi-k2.6
LLM_MODEL_HEAVY=kimi-k2.6

# 内测：关闭公开注册（员工账号由 admin 后台创建）
ALLOW_REGISTRATION=false

# 种子管理员（首次初始化用，设强密码！）
SEED_ADMIN_EMAIL=admin@你们公司.com
SEED_ADMIN_PASSWORD=一个强密码

# SMTP 内测期可不配（注册已关，不发验证邮件）
```

> ⚠️ 三个常见坑：
> 1. `DATABASE_URL` **不用填**——compose 会自动指到容器里的 postgres；
> 2. `AUTH_SECRET` 一定要新生成，不要复用开发机的；
> 3. `.env` 永远不要提交到 git。

## 第 5 步：启动！（首次约 5-10 分钟，在编译）

```
$ docker compose up -d --build
```

等它跑完，检查状态：

```
$ docker compose ps          # app 和 db 应该是 running/healthy，migrate 是 exited(0)
$ docker compose logs app | tail -20   # 应看到 Ready in xxx ms
```

**初始化种子数据（只在第一次跑）**——创建管理员账号和默认定价表：

```
$ docker compose run --rm migrate npm run db:seed
```

## 第 6 步：验收（5 分钟）

浏览器打开 **`http://你的公网IP:3000`**：

1. 用 `.env` 里的 SEED_ADMIN 邮箱+密码登录 ✓
2. 打开 `/register` 应显示「内测期间未开放注册」✓
3. 管理后台 → 用户：给自己和同事**创建账号 + 充值积分** ✓
4. 建项目 → 传剧本 → 跑一次提取，确认 LLM 通 ✓

把 `http://IP:3000` 和各自的账号密码发给同事，内测开始。

## 日常运维（收藏这几条就够了）

| 要做什么 | 命令 |
|---|---|
| 看应用日志 | `docker compose logs -f app` |
| 重启应用 | `docker compose restart app` |
| 改了 .env 后生效 | `docker compose up -d app` |
| **更新代码上线** | `cd /opt/liuguangji && git pull && docker compose up -d --build` |
| **每天备份数据库** | 见下方 ↓ |
| 停掉一切 | `docker compose down`（数据卷保留，数据不丢） |

**自动每日备份**（强烈建议第一天就做）：

```
$ mkdir -p /opt/backups
$ crontab -e     # 选 1 (nano)，在文件末尾粘贴下面这行，保存退出
0 3 * * * docker compose -f /opt/liuguangji/docker-compose.yml exec -T db pg_dump -U liuguangji liuguangji | gzip > /opt/backups/lgj-$(date +\%F).sql.gz && find /opt/backups -mtime +14 -delete
```

（每天凌晨 3 点备份，保留 14 天。恢复：`gunzip -c 备份文件 | docker compose exec -T db psql -U liuguangji liuguangji`）

## 常见问题

- **浏览器打不开**：九成是防火墙没放行 3000 端口（回第 0 步第 5 点）。
- **登录后跳回登录页**：`.env` 里 AUTH_SECRET 没设或后来改过（改了会使所有会话失效，重登即可）。
- **生成报错余额不足**：admin 后台给该用户充值积分。
- **生成慢/失败**：`docker compose logs -f app` 看报错；Kimi 偶发超时重试一次即可。
- **服务器重启后没起来**：`cd /opt/liuguangji && docker compose up -d`（已配 restart: unless-stopped，一般会自启）。
